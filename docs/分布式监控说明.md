## 🎯 核心概念

**分布式互相监控**：
```
每个 Agent 监控 2-3 个其他 Agent
    ↓
发现目标空闲 → 触发整理
    ↓
去中心化，无单点故障
    ↓
自动形成平衡态
```

**咩~ 就像**：
> 村民互相监督打扫，不需要专门的监督员

---

## 📋 工作原理

### 1. 元数据系统

每个 Agent 的记忆文件都有元数据：
```json
{
    "agentId": "main",
    "lastWrite": 1773350400000,
    "lastCleanup": 1773346800000,
    "healthScore": 85,
    "cleanupPriority": "P1",
    "monitoredBy": ["mie-mie", "es-dev-1"],
    "monitoring": ["content-director", "theater-director"]
}
```

### 2. 监控关系

```
main → 监控 → content-director, theater-director
mie-mie → 监控 → main, es-dev-1
es-dev-1 → 监控 → mie-mie, es-data-1
...
```

### 3. 触发逻辑

```
每分钟检查一次
    ↓
发现目标超过阈值没写记忆
    ↓
检查系统负载（<50% 才触发）
    ↓
检查并发数（<3 个才触发）
    ↓
触发整理
    ↓
更新健康度
```

## 🚀 使用方式

### 自动启动

记忆服务器启动时自动启动监控：
```bash
node electronic-sheep-stdio-server.js main
```

### 手动测试

```bash
# 查看元数据
node memory-meta.js getMeta main

# 查看监控关系
node memory-meta.js getMonitoringTargets main

# 手动触发整理
node shepherd-memory-idle.js main full
```
