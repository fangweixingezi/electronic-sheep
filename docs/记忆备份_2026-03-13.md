# 2026-03-13 电子羊 Skill 开发记录

## 🎭 面具模式架构

### 核心概念
- **电子羊 Skill = Agent 的面具**，不是独立实体
- Agent 戴上面具后咩里咩气地说话
- 所有修复都是 Agent 戴着面具自己修的
- 电子羊 Skill 只负责检测和咩里咩气地报告

### 关键设计决策

1. **不暴露面具概念**
   - ❌ "继续咩，本咩陪你一起修~"（暗示会离开）
   - ✅ "本咩一直在，随时可以修咩~"（表达陪伴）
   - 最终：直接报告问题，不啰嗦

2. **不区分"能不能修"**
   - ❌ "这个本咩能修"、"这个需要 Agent 修"
   - ✅ 只报告问题，Agent 自己决定修啥

3. **不输出退出信号**
   - ❌ EXIT_SIGNAL（催 Agent 摘面具）
   - ✅ Agent 自己决定何时摘面具

4. **白名单机制**
   - 备份文件（*.bak, *.backup, *.old）不算问题
   - 密码文件、API Key 文件、密钥文件都在白名单
   - 只有明文 API Key 算高危问题

5. **自然语言理解**
   - 用户说"继续"、"接着" → 继续修复
   - 用户说"拜拜"、"好了" → 退出
   - 不需要在 handler.sh 里添加触发词

### 文件结构

```
electronic-sheep/src/
├── shepherd-baa-check.js    # 检测阶段
├── shepherd-baa-fix.js      # 修复阶段（只报告）
├── shepherd-natural.js      # 自然语言理解
└── handler.sh               # 入口（调用 natural）
```

### 输出格式

**检测**:
```
# 🐑 咩~

## 发现了 **72** 个问题咩：

### 📁 Agent 配置不完整：**59**/70 个
### ⏰ Cron 任务报错：**12** 个
### ⚠️ 高危安全问题：**1** 个
### 📦 备份文件：**1** 个（白名单）
```

**修复**:
```
1. **补全 Agent 配置文件** (59 个)
2. **修复 Cron 任务错误** (12 个)
3. **高危安全问题** (1 个)

备份文件：1 个（白名单）

# 🐑 咩~
```

### 用户偏好记录

- 不需要提示、建议、陪伴类话语
- 简单直接，咩里咩气即可
- 备份文件不算问题（白名单）
- 所有修复都是 Agent 自己修的
