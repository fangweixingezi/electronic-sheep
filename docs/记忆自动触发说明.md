# 🐑 电子羊记忆系统 - 自动触发说明

**版本**: v2.7.1  
**日期**: 2026-03-13

---

## 🎯 触发机制

### ❌ 之前（Cron 定时）

```
每小时检查一次
    ↓
无论 Agent 是否空闲都检查
    ↓
浪费资源 ❌
```

### ✅ 现在（事件触发）

```
Agent 会话结束
    ↓
自动调用 autohook 脚本
    ↓
检测是否空闲（5 分钟无活动）
    ↓
空闲 → 触发记忆整理 ✅
忙碌 → 跳过 ⏸️
```

---

## 🔧 集成方式

### 在 handler.sh 中添加

```bash
# Agent 会话结束时调用
session_end() {
    local agentId="$1"
    
    # 触发记忆自动整理
    node "$SCRIPT_DIR/shepherd-memory-autohook.js" "$agentId"
}
```

### 或在 OpenClaw 配置中

```json
{
  "hooks": {
    "sessionEnd": "node ~/.openclaw/agents/main/skills/electronic-sheep/src/shepherd-memory-autohook.js {agentId}"
  }
}
```

## 📊 资源对比

| 方式 | 触发频率 | 资源占用 | 效率 |
|------|---------|---------|------|
| **Cron 定时** | 每小时 1 次 | 高（无效检查多） | ❌ 低 |
| **事件触发** | 会话结束时 | 低（只在空闲时） | ✅ 高 |

---

## 🎯 使用场景

### 适合触发记忆整理的时机

1. ✅ Agent 完成一次对话
2. ✅ Agent 完成一个任务
3. ✅ Agent 会话超时
4. ✅ Agent 主动进入休息

### 不适合触发的时机

1. ❌ Agent 正在处理任务
2. ❌ Agent 连续对话中
3. ❌ Agent 刚被唤醒
