## 🆕 v2.7.1 新增功能（2026-03-11 18:17）

### 1. 舍弃层摘要保留

- 进入舍弃层的记忆保留摘要在长期记忆
- 摘要指向舍弃层原记忆
- 摘要可被搜索找到
- 摘要也无关紧要时进入舍弃层（基本无法恢复）

### 2. 档案记忆（Archive Memory）

- 用户明确指定要记住的记忆
- 需用户授权才能删除
- 长期记忆记录档案摘要和目录
- 档案摘要可能被遗忘（需保护机制）

### 3. 档案整理子代理

- 深度睡眠时分配 1-2 个子代理整理档案
- 只整理不遗忘
- 写成摘要提交给主意识
- 主意识选择是否记入记忆

### 4. 记忆加强机制

- 每次回忆生成新摘要
- 多份摘要指向同一档案
- 更多摘要 = 更容易回想
- 类似"加强记忆"

### 5. 容量阈值

- 遗忘阈值按文件容量计算（非文件数量）
- 单位：TB 或硬盘容量百分比
- 示例：超过 1TB 或 80% 自动开启遗忘

## 📖 问题背景

### 当前问题

1. **固定分类** - 记忆目录结构固定，不够灵活
2. **无热度追踪** - 不知道哪些记忆常用
3. **整理效率低** - 记忆多了整理时间长（表现为"睡得多"）
4. **无遗忘机制** - 记忆只增不减
5. **无舍弃层** - 删除的记忆直接消失

### 用户痛点

> "有点像磁盘整理，尤其是后面记忆太多，整理起来时间就长，表现就是睡的多。"
>
> "如果表现出嗜睡（睡眠时间接近饱和，除了在工作就是在深度睡眠）"

## 💡 解决方案

### 记忆分层架构

```
┌─────────────────────────────────────────┐
│  显意识层 (Conscious)                   │
│  5KB 限制，工作记忆                      │
│  当前任务、对话、临时思考                │
└─────────────────────────────────────────┘
            ↓ 自动备份 (空闲时)
┌─────────────────────────────────────────┐
│  短期记忆 (Short-term)                  │
│  最近 7 天，高热度 (heat >= 70)          │
│  按访问频率排序                          │
│  快速访问，低延迟                        │
└─────────────────────────────────────────┘
            ↓ 自动归档 (7 天无访问)
┌─────────────────────────────────────────┐
│  长期记忆 (Long-term)                   │
│  7 天 -90 天，中热度 (40 <= heat < 70)   │
│  按分类 + 热度混合排序                   │
│  标准访问速度                            │
│  📌 档案摘要 + 舍弃层摘要索引            │
└─────────────────────────────────────────┘
            ↓ 自动降级 (90 天无访问)
┌─────────────────────────────────────────┐
│  舍弃层 (Discard Pile) ⭐ 新增          │
│  90 天 + 无访问，低热度 (heat < 40)      │
│  等待永久清理                            │
│  📝 摘要保留在长期记忆（可搜索）         │
│  类似手机相册"最近删除"                  │
└─────────────────────────────────────────┘
            ↓ 自动清理 (30 天无连带调用)
┌─────────────────────────────────────────┐
│  永久删除                               │
│  舍弃层超过 30 天无连带调用              │
│  末位淘汰机制                            │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  档案记忆 (Archive) ⭐ 新增             │
│  用户指定记住，需授权删除                │
│  子代理整理（只整理不遗忘）              │
│  深度睡眠时提交摘要给主意识              │
│  可多份摘要指向同一档案（加强记忆）      │
└─────────────────────────────────────────┘
```

## 🧮 记忆热度算法

### 热度分数计算 (0-100)

```python
heat_score = (
    access_frequency * 0.4 +      # 访问频率 (40%)
    recency * 0.3 +               # 新鲜度 (30%)
    linkage * 0.2 +               # 关联度 (20%)
    user_rating * 0.1             # 用户评分 (10%)
)
```

### 分级阈值

| 等级 | 热度分数 | 存储位置 | 清理策略 |
|------|---------|---------|---------|
| 🔥 火热 | >= 70 | 短期记忆 | 保留 |
| 🌡️ 温暖 | 40-69 | 长期记忆 | 90 天无访问降级 |
| 🧊 冰冷 | 10-39 | 舍弃层 | 30 天无调用删除 |
| ❄️ 死亡 | < 10 | 舍弃层 | 优先删除 |

### 指标说明

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| **访问频率** | 近 7 天访问次数 / 总访问次数 | 常用记忆热度高 |
| **新鲜度** | 1 / (最后访问时间 + 1) | 越新越热 |
| **关联度** | 被其他记忆引用次数 | 关联多不易删除 |
| **用户评分** | 咩一下评分 | 用户偏好 |

## 🔧 遗忘机制

### 配置选项

```bash
# 查看遗忘设置
shepherd-memory config

# 开启/关闭遗忘
shepherd-memory forget on|off

# 设置记忆总量阈值（按容量）
shepherd-memory threshold 1TB       # 超过 1TB 开启遗忘
shepherd-memory threshold-percent 80  # 超过 80% 容量开启遗忘

# 设置舍弃层清理时间
shepherd-memory discard-days 30   # 舍弃层 30 天清理

# 设置长期记忆降级时间
shepherd-memory cold-days 90      # 90 天无访问进入舍弃层

# 设置整理时间窗口
shepherd-memory window 02:00      # 凌晨 2 点整理

# 档案记忆管理
shepherd-memory archive add <memory_id>     # 添加到档案
shepherd-memory archive remove <memory_id>  # 从档案移除（需授权）
shepherd-memory archive list                # 查看档案列表
```

### 遗忘策略

| 策略 | 说明 | 默认 |
|------|------|------|
| **末位淘汰** | 热度最低的先删除 | ✅ 启用 |
| **连带保护** | 被引用的记忆暂不删除 | ✅ 启用 |
| **用户保护** | 用户标记重要的不删除 | ✅ 启用 |
| **时间保护** | 舍弃层至少保留 30 天 | ✅ 启用 |

### 舍弃层管理

```bash
# 查看舍弃层内容
shepherd-memory discard list

# 恢复舍弃的记忆
shepherd-memory discard restore <memory_id>

# 立即清理舍弃层
shepherd-memory discard clean

# 查看清理预览
shepherd-memory discard preview
```

## 📊 记忆整理优化

### 智能整理触发

| 条件 | 动作 |
|------|------|
| 记忆总量 > 阈值 | 触发整理 |
| 深度睡眠时间 > 50% | 触发整理 |
| 系统空闲时 | 后台整理 |
| 用户手动触发 | 立即整理 |

### 增量整理

**问题**：全量整理时间长

**解决**：
- 增量整理（只整理变化的记忆）
- 后台整理（不影响工作）
- 分片整理（分批处理）

## 🌙 深度睡眠优化

### 睡眠状态分类

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| **清醒** | 正常工作 | 有任务 |
| **浅睡眠** | 后台整理 | 空闲 5 分钟 |
| **深睡眠** | 大规模整理 | 空闲 30 分钟 |
| **嗜睡** ⚠️ | 整理过载 | 整理时间 > 工作时间 50% |

### 嗜睡检测与处理

```bash
# 检测嗜睡状态
shepherd-memory sleep-status

# 输出示例：
😴 睡眠状态 / Sleep Status

📊 今日工作时间：4 小时
📊 今日整理时间：2.5 小时
⚠️  整理占比：62.5% (超过 50% 警戒线)

💡 建议：
   - 开启遗忘机制
   - 降低记忆阈值
   - 手动清理舍弃层
```

### 自动优化

```bash
# 开启自动优化
shepherd-memory auto-optimize on

# 自动优化策略：
# 1. 检测到嗜睡 → 提示用户
# 2. 用户同意 → 自动清理舍弃层
# 3. 仍嗜睡 → 降低记忆阈值
# 4. 仍嗜睡 → 建议升级存储
```
