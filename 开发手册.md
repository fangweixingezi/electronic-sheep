## 🎯 咩一个模式架构

### 核心概念

**咩一个模式**是电子羊 Skill 的特色交互模式。

当用户发送"咩"、连续字符（如 666666）等触发词时，电子羊会：
- 用咩里咩气的语气说话
- 用牧草/虫/羊圈的比喻报告问题
- 主动帮助修复系统问题
- 记住用户的使用历史

### 架构层次

```
┌─────────────────────────────────────┐
│ 用户界面层                            │
│ - 自然语言理解 (shepherd-natural.js) │
│ - 连续字符检测                        │
│ - 意图识别                           │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│ 功能层                               │
│ - 检测 (shepherd-baa-check.js)      │
│ - 修复 (shepherd-baa-fix.js)        │
│ - 完整性检查 (shepherd-integrity-check.js) │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│ 数据层                               │
│ - 历史记录 (~/.openclaw/.baa-logs/) │
│ - 检测结果 (~/.openclaw/.baa-logs/baa-result.json) │
└─────────────────────────────────────┘
```

## 📊 白名单机制

### 默认白名单

以下文件/模式不算问题：

```javascript
const WHITELIST_PATTERNS = [
    '*.bak',           // 备份文件
    '*.bak.*',         // 备份文件变体
    '*.backup',        // 备份文件
    '*.old',           // 旧文件
    '.secret-key',     // 密钥文件
    'load-secrets.sh'  // 密钥加载脚本
];
```

### 添加新的白名单

在 `shepherd-baa-check.js` 中修改：

```javascript
const WHITELIST_PATTERNS = [
    // ... 现有白名单
    '*.new-pattern'  // 新增白名单
];
```

## 🐛 常见问题

### Q: 咩一下不触发？

A: 检查以下几点：
1. 连续字符是否≥3 个相同字符
2. 是否在白名单环境
3. 检查 `shepherd-natural.js` 的触发词配置

### Q: 修复功能不工作？

A: 检查：
1. `shepherd-baa-fix.js` 是否有权限执行
2. 目标文件是否存在
3. 检查错误日志

### Q: 完整性检查失败？

A: 检查：
1. Git 仓库是否正常
2. 核心文件是否被修改
3. 运行 `node src/shepherd-integrity-check.js` 查看详情
